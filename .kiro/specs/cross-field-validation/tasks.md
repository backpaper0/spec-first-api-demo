# 実装計画

## タスク一覧

- [x] 1. `@ValidDateRange` 日付範囲バリデーションを実装する
- [x] 1.1 日付範囲制約アノテーションを定義する
  - `from` と `to` 属性（文字列でフィールド名を指定）および `message`・`groups`・`payload` の Bean Validation 標準属性を持つクラスレベルアノテーションを作成する
  - 同一クラスへの複数アノテーション指定（`@Repeatable`）をサポートするコンテナアノテーションも合わせて定義する
  - `common/validation` パッケージを新設してここに配置する
  - _Requirements: 1.6, 1.7, 3.3_

- [x] 1.2 日付範囲バリデーターのロジックを実装する
  - クラスレベルの `ConstraintValidator` としてアノテーション初期化時に `from`・`to` のフィールド名を保持する
  - Bean からリフレクションで各フィールドの値を取得し、`null` または空文字の場合はチェックをスキップして検証を通過させる
  - `uuuu/MM/dd` フォーマットかつ厳密モードで日付文字列をパースし、フォーマット不正の場合はバリデーションエラーとする
  - `from` 日付が `to` 日付より後の場合にバリデーションエラーを返す（同日は有効）
  - エラー時はデフォルトの違反を無効化し、`to` フィールドに紐付けた `ConstraintViolation` を生成する
  - 存在しないフィールド名が指定された場合はバリデーション例外でフェイルファストする
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 3.1, 3.4_

- [x] 1.3 日付範囲バリデーターの単体テストを作成する
  - 有効ケース: `from` < `to`、`from` = `to`（境界値）
  - エラーケース: `from` > `to`、不正フォーマット（ハイフン区切り、存在しない月日など）
  - スキップケース: `from` が null、`to` が空文字
  - クラス行カバレッジ 80% 以上を達成する（JaCoCo 要件）
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 2. `@RequiredWhen` 条件付き必須バリデーションを実装する
- [x] 2.1 (P) 条件付き必須制約アノテーションを定義する
  - タスク 1.1 と並行して実施可能
  - `field`（必須チェック対象フィールド名）と `dependsOn`（条件となる依存フィールド名）属性および `message`・`groups`・`payload` の Bean Validation 標準属性を持つクラスレベルアノテーションを作成する
  - 同一クラスへの複数アノテーション指定（`@Repeatable`）をサポートするコンテナアノテーションも合わせて定義する
  - `common/validation` パッケージに配置する
  - _Requirements: 2.4, 2.5, 3.3_

- [x] 2.2 (P) 条件付き必須バリデーターのロジックを実装する
  - タスク 2.1 完了後、タスク 1.2 と並行して実施可能
  - クラスレベルの `ConstraintValidator` としてアノテーション初期化時に `field`・`dependsOn` のフィールド名を保持する
  - Bean からリフレクションで `dependsOn` フィールドの値を取得し、`null` または空文字の場合はチェックをスキップして検証を通過させる
  - `dependsOn` に値がある場合は `field` フィールドの値を取得し、`null` または空文字であればバリデーションエラーを返す
  - エラー時はデフォルトの違反を無効化し、`field` フィールドに紐付けた `ConstraintViolation` を生成する
  - `field` に `String` 以外の型のフィールドが指定された場合はバリデーション例外でフェイルファストする
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.4_

- [x] 2.3 (P) 条件付き必須バリデーターの単体テストを作成する
  - タスク 2.2 完了後、タスク 1.3 と並行して実施可能
  - 有効ケース: `dependsOn` に値あり ＆ `field` に値あり
  - エラーケース: `dependsOn` に値あり ＆ `field` が null、`dependsOn` に値あり ＆ `field` が空文字
  - スキップケース: `dependsOn` が null、`dependsOn` が空文字
  - クラス行カバレッジ 80% 以上を達成する（JaCoCo 要件）
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 3. Spring MVC との統合テストを実装する
  - タスク 1・2 の完了後に実施する（両バリデーター実装が前提）
  - `@WebMvcTest` を使ってテスト用コントローラと Request Bean を作成し、`@Valid` によるバリデーション発火を確認する
  - `@ValidDateRange` を付与した Bean に対して不正な日付範囲のリクエストを送信し、400 レスポンスが返ることおよびエラーが `to` フィールドに紐付くことを検証する
  - `@RequiredWhen` を付与した Bean に対して依存フィールドあり・必須フィールドなしのリクエストを送信し、400 レスポンスが返ることおよびエラーが `field` フィールドに紐付くことを検証する
  - _Requirements: 3.2_
